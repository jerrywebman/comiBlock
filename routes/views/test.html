Welcome Bro, These are the existing API Endpoints Base Route: https://comiblock.herokuapp.com/
Route Action Request.body/params
SIGNUP, LOGIN, LOGOUT, CONFIRM EMAIL, RECOVER ACCOUNT
GET-/logout Logout -
POST-/api/signup Signup - req.body.phone, req.body.firstname, req.body.surname, req.body.email, req.body.occupation, req.body.dateOfBirth, req.body.password

By default a verified state is set to false, and the user is sent an OTP to confirm email address.
POST-/api/recover_account Recover Account req.body.email

Sends a new OTP to the user email address
POST-/api/confirm_email confirm Email address req.body.email, req.body.verifyCode

Verifies the OTP sent to the user email address with what is stored in the DB and updates the user verified status to true if correct
PATCH-/api/update/password Update Password req.body.email,req.body.confirmPassword,req.body.newPassword
POST-/api/login Authentication/login req.body.email, req.body.password
GET-/api/getinfo Response with auth user details res.json({});
Risk Tolerance and Crypto Exposure assessment
POST-/api/user/create/assessment Creates User assessment data(authenticated route) { question: req.body.assessmentOneQ, answer: req.body.assessmentOneA, }, { question: req.body.assessmentTwoQ, answer: req.body.assessmentTwoA, }, { question: req.body.assessmentThreeQ, answer: req.body.assessmentThreeA, }, { question: req.body.assessmentFourQ, answer: req.body.assessmentFourA, }, { question: req.body.riskOneQ, answer: req.body.riskOneA, }, { question: req.body.riskTwoQ, answer: req.body.riskTwoA, }, { question: req.body.riskThreeQ, answer: req.body.riskThreeA, }, { question: req.body.riskFourQ, answer: req.body.riskFourA, },
GET-/api/user/get/assessment Responds with all user responses res.json({});
Portfolio
POST-/api/portfolio/create Creates comic block Portfolio name: req.body.name, intro: req.body.intro, briefing: req.body.briefing, strategySnapshot: req.body.strategySnapshot, holdings: [req.body.btc, req.body.eth, req.body.usdt], portfolioStrategy: req.body.portfolioStrategy, portfolioType: req.body.portfolioType, subscription: req.body.subscription, subscriptionAmount: req.body.subscriptionAmount,
GET-/api/portfolio Responds with all portfolio res.json([]);
GET-/api/portfolio/:id GETS a portfolio req.params.id, res.json({});
DELETE-/api/portfolio/:id DEL a portfolio req.params.id
Upload selfie and id card
POST-/api/upload uploads both govtId and a selfie file to the comiblock db pass two file input fields with name "profiles". by default the first one will be the govtId and the second will be the selfie.
